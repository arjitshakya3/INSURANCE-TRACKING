# Insurance Renewal Advisor

Offline-first Android app for vehicle insurance advisors. The app is written as a Flutter project and targets Android.

## Features

- Mobile OTP login with a demo local OTP flow.
- Optional app PIN lock.
- Dashboard counts for total vehicles, active policies, expired policies, and policies expiring in 15 days.
- Vehicle and policy entry form with date pickers, policy type dropdown, PI toggle, premium, reference contact, and agent name.
- RC, insurance policy, and KYC uploads as image/PDF files copied into app-local storage.
- SQLite offline storage.
- Daily local notification registration for policies expiring in the next 15 days.
- Search by vehicle number or customer name.
- Filters for expiring soon, active, expired, and insurance company.
- PDF export and Excel-compatible `.xls` export.
- JSON backup and restore.
- WhatsApp and SMS reminder actions from each policy card.
- Dark mode.

## Build

This workspace did not have Flutter, Java, Android SDK, Gradle, or ADB available on PATH during creation, so the APK could not be built locally here.

On a machine with Flutter installed:

```powershell
flutter pub get
flutter build apk --debug
```

The APK will be created under:

```text
build/app/outputs/flutter-apk/app-debug.apk
```

If the Android wrapper files are regenerated by your Flutter version, run:

```powershell
flutter create --platforms android .
```

Then reapply the package name `com.advisor.insurance_renewal` if Flutter rewrites Android metadata.

## Emulator QA Plan

After the Android toolchain is available:

```powershell
adb devices
flutter install
adb -s <serial> shell monkey -p com.advisor.insurance_renewal 1
adb -s <serial> exec-out uiautomator dump /dev/tty > artifacts/ui-dashboard.xml
adb -s <serial> exec-out screencap -p > artifacts/dashboard.png
adb -s <serial> logcat -d > artifacts/logcat.txt
adb -s <serial> shell dumpsys gfxinfo com.advisor.insurance_renewal > artifacts/gfxinfo.txt
adb -s <serial> shell dumpsys meminfo com.advisor.insurance_renewal > artifacts/meminfo.txt
```

The focused QA flow should cover login, adding a policy with an end date inside 15 days, verifying the red dashboard highlight, exporting, opening WhatsApp/SMS actions, and relaunching to confirm SQLite persistence.
